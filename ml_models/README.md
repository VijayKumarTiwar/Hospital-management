# ML Prediction Code (standalone export)

This is the `prediction` app from the hospital backend, containing just the
machine learning pieces: training script, inference wrapper, trained model
files, and the Django serializers/views/models that expose them as an API.

## Files

| File | Purpose |
|---|---|
| `train_models.py` | Generates synthetic training data, trains logistic regression pipelines for diabetes & heart disease risk, saves them as `.joblib` |
| `ml.py` | Loads the saved models (cached) and exposes `predict_diabetes()` / `predict_heart_disease()` |
| `ml_models/*.joblib` | Pre-trained model files (already generated — ready to use) |
| `models.py` | Django model `PredictionRecord` — logs every prediction made |
| `serializers.py` | Input validation for the two prediction endpoints |
| `views.py` | DRF views: `DiabetesRiskView`, `HeartDiseaseRiskView`, `PredictionHistoryViewSet` |
| `urls.py` | Routes: `/diabetes/`, `/heart-disease/`, `/history/` |

## Using just the ML part (no Django required)

`ml.py` and `train_models.py` have no Django dependency — only
`scikit-learn`, `pandas`, `numpy`, and `joblib`. You can use them standalone:

```bash
pip install scikit-learn pandas numpy joblib

# Retrain from scratch (optional — pre-trained files are already in ml_models/)
python train_models.py
```

```python
from ml import predict_diabetes, predict_heart_disease

result = predict_diabetes({
    "age": 45, "bmi": 31.2, "glucose": 145,
    "blood_pressure": 85, "family_history": 1,
})
print(result)
# {'risk_score': 0.9978, 'risk_label': 'High', 'features_used': [...]}

result = predict_heart_disease({
    "age": 55, "cholesterol": 240, "resting_bp": 140,
    "max_heart_rate": 130, "smoker": 1, "diabetic": 0,
})
print(result)
```

## Re-plugging into a Django project

Drop this whole `prediction/` folder into a Django project's app directory,
add `"prediction"` to `INSTALLED_APPS`, include `prediction.urls` in your
root `urls.py`, and run `python manage.py makemigrations prediction &&
python manage.py migrate`.

## Important caveat

These models are trained on **synthetic data** generated by a hand-built
logistic formula (see `_make_diabetes_dataset` / `_make_heart_dataset` in
`train_models.py`) — they're realistic enough for demoing the pipeline but
are **not validated on real patients** and must not be used for actual
clinical decisions. Replace with models trained on real, properly licensed
datasets (e.g. Pima Indians Diabetes, UCI Heart Disease) before any
non-prototype use.
